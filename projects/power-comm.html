超级电容与无线充电系统

围绕双向 Buck-Boost 超级电容控制器与 LCC-S 无线充电系统展开，完成能量管理、动能回收、ASK 通讯与 120W 级无线充电联调。

背景与目标

在新赛季能量限制规则下，机器人系统不再只是面对简单的功率约束，而是需要围绕“能量获取、能量存储、能量回收和能量调度”做整体设计。超级电容系统需要在裁判系统限制内完成高响应能量调配，无线充电系统则需要在补能阶段高效、安全地向电容组充能。

本项目参考《RM2025 超级电容与无线充电系统技术方案》的架构思路，对双向 Buck-Boost 超级电容控制器、峰值电流内环、数字外环、ASK / FSK 通讯和 LCC-S 无线充电链路进行整理，并将其压缩成适合个人项目展示的工程化页面。

我的职责

  • 围绕超级电容控制与无线充电系统梳理硬件拓扑、控制链路和保护逻辑；
  • 整理双向 Buck-Boost 功率级、峰值电流控制与数字外环的核心实现思路；
  • 归纳无线充电发射端、接收端和 ASK / FSK 通讯链路的协同方式；
  • 提炼关键问题闭环，用项目化表达呈现系统设计、调试与保护策略。

技术方案

超级电容部分采用双向四开关 Buck-Boost 架构，使电容组电压可以高于、等于或低于底盘电压；无线充电部分采用全桥逆变 + LCC-S 谐振拓扑，发射端和接收端通过 ASK / FSK 方式完成握手与功率反馈。两套系统最终汇总到同一套能量管理逻辑中。

// 超级电容控制主链路
read_adc();                         // 采样 vA / vB / iA / iB / iR
mode = select_mode_by_voltage();   // Buck / Buck-Boost / Boost
peak_i_ref = outer_loop_control(); // 功率环 / 电压环 / 电流限制环竞争输出
set_dac_threshold(peak_i_ref);     // 峰值电流内环阈值

// 无线充电主链路
tx_full_bridge();                  // 发射端全桥逆变 + LCC-S 谐振
rx_buck_charge();                  // 接收端降压给电容组充电
ask_feedback(power, enable_bit);   // 反馈接收功率与开关需求

控制上最关键的设计是“峰值电流内环 + 数字外环”。峰值电流内环利用 STM32G474 的 HRTIM、比较器、DAC 与高速模拟外设在硬件层快速限制电感电流，数字外环则以约 62.5kHz 运行裁判系统功率环、电容组电压环和电流限制环，最终共同决定目标电流。

无线充电部分则重点处理发射端功率传输、接收端 Buck 充电、ASK 解码和状态机联动，使系统在补能阶段可以以 120W 级功率安全充入电容组，并在异常状态下及时关断输出。

遇到的问题与解决

问题 1：纯数字控制方案响应速度难以继续提升。仅靠软件环路提高响应速度会受到 MCU 运算负担和采样带宽限制，环路一旦提得太快，其他保护与交互逻辑就容易被挤压。为此，系统引入峰值电流内环，将电感电流控制尽量下沉到模拟外设和 HRTIM 硬件层，使电流响应进入 20us 级别，同时释放 MCU 资源给外环与保护逻辑。

问题 2：Buck / Buck-Boost / Boost 模式切换容易产生抖动。占空比靠近边界时，模式如果切换过于频繁会造成电流波动和控制不连续。报告中通过电压比估算占空比，并在状态机中加入施密特触发和模式冗余区间，我在页面中保留了这套思路，用于解释为何不同模式切换能够保持平滑。

问题 3：上管常开若依赖隔离电源会带来较高静态功耗。传统做法实现简单，但静态损耗偏高，不利于能量利用。该方案通过类似电荷泵的方式实现上管常开，显著降低静态功耗，并减少额外器件占用，这也是整套硬件方案的一个亮点。

问题 4：电容低压区放电能力受限，若控制不当会导致系统不稳定。电容组在低电压时虽然仍能输出能量，但功率能力会快速下降。为此系统将低压限制与放电截止做成独立约束，并让电容电压环、电流限制环和功率环竞争输出，保证在低压区优先保护系统本身。

问题 5：无线充电链路不仅要传能，还要可靠通讯。未对准、充电功率变化和线圈电压幅值波动都会影响通信解码。方案中通过 ASK / FSK 双链路设计、发射端状态机和接收端功率反馈，使发射端能够根据接收端功率与开关需求动态调整输出，避免盲目持续充电。

问题 6：保护系统需要足够快，否则短路和过压风险极高。超级电容与无线充电系统都涉及大功率能量流，一旦短路或失控会迅速损坏功率级。方案利用 ADC 模拟看门狗、HRTIM Fault Line、短路计数器和错误分级机制实现分层保护，在短路测试中能够快速关断，减少火花和损伤风险。

问题 7：无线充电与超级电容系统融合后，异常工况变得更复杂。例如底盘断电、无线充电仍在工作、或接口短路时，都可能造成保险丝熔断或器件损坏。文档中对这些未来改进方向有明确记录，我在页面中保留这些工程风险说明，用来体现系统设计对极端场景的考虑。
